# MovieMatch 🎬

**MovieMatch** is a technical playground built to master the **Modern Android Development (MAD)** stack. The goal is to transform a classic movie catalog integration into a cutting-edge reactive application.

---

## 🚀 About the Project

This repository serves as a sandbox to explore and implement the latest industry standards. Starting from a standard use case—displaying a movie catalog via the TMDB API—it evolves into a full-scale showcase of modern patterns and libraries.

### Key Features & Learning Goals

* **Modern UI with Compose:** Implementing a fully declarative interface with a focus on polished UX, custom themes, and smooth transitions.
* **Reactive Data Streams:** Leveraging **Kotlin Flow** and **Coroutines** for robust, lifecycle-aware asynchronous data handling.
* **TMDB Integration:** Fetching real-world data from the TMDB API using **Retrofit** and handling complex pagination.
* **Edge-of-the-stack Navigation:** Exploring the latest **Navigation 3 API** to manage screen flows and state-based routing.
* **Clean Architecture:** Maintaining a scalable codebase with a clear separation of concerns (Data, Domain, and UI layers).

---

## 🛠 Tech Stack

* **Language:** [Kotlin](https://kotlinlang.org/)
* **UI:** [Jetpack Compose](https://developer.android.com/jetpack/compose)
* **Navigation:** Navigation 3 (Experimental/Alpha)
* **Asynchronous:** Coroutines & Flow
* **Dependency Injection:** Hilt
* **Networking:** Retrofit & OkHttp
* **Image Loading:** Coil
* **Architecture:** MVVM + Clean Architecture
* **Local Database :** **Room** (for offline caching).
* **Testing :** MockK, JUnit 5, and Compose Testing Library.
